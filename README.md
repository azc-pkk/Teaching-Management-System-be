# 教学过程管理系统

本仓库用于小组完成“教学过程管理系统”选题的 TypeScript 全栈开发。项目建议采用 `Next.js + TypeScript + PostgreSQL + Prisma`，前后端在同一仓库内协作。

## 先读文档

开发前请先阅读：

- [教学过程管理系统功能设计.md](./教学过程管理系统功能设计.md)

该文档包含项目背景、角色权限、核心业务模块、前后端任务拆分、API 规范、数据模型草案、里程碑和验收清单。

## 分工建议

- 前端工程师：负责页面路由、后台布局、表格表单、审批流页面、统计视图、移动端适配。
- 后端工程师：负责数据库模型、登录鉴权、REST API、业务状态流转、导入导出、接口测试。
- 项目经理：负责需求范围控制、任务拆分、进度检查、功能验收和文档维护。

## 开发顺序

1. 搭建 Next.js 项目基础结构。
2. 完成登录、角色权限、菜单布局。
3. 完成教师、学生、课程、教室、班级等基础数据管理。
4. 完成教室申请、调课审批、考务安排三个核心流程。
5. 补充教材、毕业审核、教学日志、统计报表等模块。
6. 联调、测试、准备演示数据和答辩材料。

## 协作约定

- 每次开始开发前先执行 `git pull`，避免和组员代码冲突。
- 每个功能尽量单独提交，提交信息写清楚做了什么。
- 不要直接删除他人代码；遇到冲突先沟通再处理。
- 接口字段以 TypeScript 类型和后端返回为准，前后端保持一致。
- 外部系统如 CAS、鹏达系统、指纹考勤、短信平台第一版先用 mock 或预留接口。
