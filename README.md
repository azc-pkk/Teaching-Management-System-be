# 教学过程管理系统

本仓库用于小组完成“教学过程管理系统”选题的 TypeScript 全栈开发。项目建议采用 `Next.js + TypeScript + PostgreSQL + Prisma`，并使用 Apifox 管理接口文档、Mock 数据和接口测试，前后端在同一仓库内协作。

## 先读文档

开发前请先阅读：

- [教学过程管理系统功能设计.md](./教学过程管理系统功能设计.md)

该文档包含项目背景、角色权限、核心业务模块、前后端任务拆分、API 规范、数据模型草案、里程碑和验收清单。

## 分工建议

- 前端工程师：负责页面路由、后台布局、表格表单、审批流页面、统计视图、移动端适配，并根据 Apifox 接口文档和 Mock 数据开发页面。
- 后端工程师：负责数据库模型、登录鉴权、REST API、业务状态流转、导入导出、接口测试，并及时维护 Apifox 接口文档。
- 项目经理：负责需求范围控制、任务拆分、进度检查、功能验收和文档维护。

## 开发顺序

1. 搭建 Next.js 项目基础结构。
2. 在 Apifox 建立项目接口目录、统一响应格式、公共环境变量和 Mock 数据。
3. 完成登录、角色权限、菜单布局。
4. 完成教师、学生、课程、教室、班级等基础数据管理。
5. 完成教室申请、调课审批、考务安排三个核心流程。
6. 补充教材、毕业审核、教学日志、统计报表等模块。
7. 通过 Apifox 联调接口并补充测试用例，准备演示数据和答辩材料。

## 协作约定

- 每次开始开发前先执行 `git pull`，避免和组员代码冲突。
- 每个功能尽量单独提交，提交信息写清楚做了什么。
- 不要直接删除他人代码；遇到冲突先沟通再处理。
- 接口字段以 Apifox 文档、TypeScript 类型和后端返回为准，三者必须保持一致。
- 新增或修改接口时，先更新 Apifox，再开发后端和前端调用。
- 前端可以先使用 Apifox Mock 地址开发，后端完成后再切换真实接口环境。
- 后端接口提测前，需要在 Apifox 中补充成功、失败、越权等测试用例。
- 外部系统如 CAS、鹏达系统、指纹考勤、短信平台第一版先用 mock 或预留接口。
