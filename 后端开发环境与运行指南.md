# 后端开发环境与运行指南

本文档给后端开发员使用，说明本地环境配置、Docker MySQL、Prisma 和 NestJS 后端运行流程。

## 1. 环境要求

- Node.js 20 或更高版本
- pnpm 10 或更高版本
- Docker Desktop，启动后确认 Docker Engine 正常运行
- Git
- 推荐编辑器：Visual Studio Code

检查命令：

```bash
node -v
pnpm -v
docker --version
docker compose version
```

## 2. 项目初始化

进入后端仓库：

```bash
cd E:\clone\Teaching-Management-System-be
```

安装依赖：

```bash
pnpm install
```

复制环境变量文件：

```bash
copy .env.example .env
```

本项目默认使用 Docker MySQL，连接串如下：

```env
DATABASE_URL="mysql://teaching_user:teaching_password@localhost:3307/teaching_management"
```

说明：本机 3306 端口可能已有 MySQL，因此 Docker MySQL 映射到宿主机 3307，容器内部仍是 3306。

## 3. 启动 Docker MySQL

启动数据库容器：

```bash
docker compose up -d
```

查看容器状态：

```bash
docker compose ps
```

正常状态应看到 `mysql` 服务为 `healthy`，端口类似：

```text
0.0.0.0:3307->3306/tcp
```

停止容器：

```bash
docker compose down
```

如果需要删除数据库数据并重新初始化：

```bash
docker compose down -v
docker compose up -d
```

## 4. Prisma 数据库命令

修改 `prisma/schema.prisma` 后，先生成 Prisma Client：

```bash
pnpm prisma:generate
```

首次初始化或同步已有迁移：

```bash
pnpm exec prisma migrate deploy
```

开发新表结构时创建迁移：

```bash
pnpm prisma:migrate -- --name 迁移名称
```

写入演示数据：

```bash
pnpm exec prisma db seed
```

打开数据库可视化：

```bash
pnpm prisma:studio
```

Prisma Client 生成到 `generated/prisma`，该目录是生成物，不提交到仓库。拉取新代码后如果类型报错，先运行：

```bash
pnpm prisma:generate
```

## 5. 启动后端服务

开发模式启动：

```bash
pnpm start:dev
```

后端默认地址：

```text
http://localhost:3001
```

健康检查：

```text
http://localhost:3001/api/health
```

看到下面结构即表示后端启动成功：

```json
{
  "success": true,
  "data": {
    "service": "teaching-management-system-be",
    "status": "ok"
  }
}
```

## 6. 推荐完整启动顺序

每天开始开发时：

```bash
cd E:\clone\Teaching-Management-System-be
docker compose up -d
pnpm prisma:generate
pnpm exec prisma migrate deploy
pnpm start:dev
```

第一次配置或需要重置演示数据时：

```bash
docker compose up -d
pnpm prisma:generate
pnpm exec prisma migrate deploy
pnpm exec prisma db seed
pnpm start:dev
```

## 7. 常见问题

### Docker 命令不可用

先启动 Docker Desktop，再执行：

```bash
docker info
```

如果仍失败，检查 Docker Desktop 是否完成初始化。

### 3306 端口被占用

本项目已经把 Docker MySQL 映射到 3307，一般不需要停止本机 MySQL。确认 `.env` 使用的是：

```env
DATABASE_URL="mysql://teaching_user:teaching_password@localhost:3307/teaching_management"
```

### Prisma Client 找不到

运行：

```bash
pnpm prisma:generate
```

不要手动修改 `generated/prisma` 下的文件。

### seed 执行失败

先确认数据库迁移已完成：

```bash
pnpm exec prisma migrate status
pnpm exec prisma migrate deploy
```

再运行：

```bash
pnpm exec prisma db seed
```

### 浏览器拒绝访问后端

确认后端已启动，并访问完整路径：

```text
http://localhost:3001/api/health
```

如果端口未监听，重新运行：

```bash
pnpm start:dev
```

## 8. 提交前检查

提交代码前至少运行：

```bash
pnpm typecheck
pnpm test
pnpm build
```

数据库结构变更必须提交：

- `prisma/schema.prisma`
- `prisma/migrations/**`

不要提交：

- `.env`
- `node_modules`
- `generated/prisma`
- `dist`
- `package-lock.json`
