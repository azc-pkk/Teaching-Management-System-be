# 登录注册前端 UI 说明

本项目的注册功能只面向学生开放。学生必须先使用“姓名 + 学号”匹配系统中已有的学生档案，匹配成功后设置自己的登录密码。教师、管理员、系部管理员、教材管理员、领导等角色不允许在前台自助注册，这些账号由系统管理员在后台维护。

## 页面建议

建议前端把登录页设计成一个认证页面，包含两个 Tab：

- 登录
- 学生注册

### 登录 Tab

字段：

| 字段 | 说明 |
| --- | --- |
| 账号 | 管理员、教师使用系统分配账号；学生注册后使用学号 |
| 密码 | 用户密码 |

按钮：

```text
登录
```

请求接口：

```http
POST /api/auth/login
```

请求体：

```json
{
  "username": "20230004",
  "password": "Student123456"
}
```

登录成功后保存：

```text
data.token
data.user
```

### 学生注册 Tab

字段：

| 字段 | 说明 |
| --- | --- |
| 学号 | 必填，必须存在于学生档案中 |
| 姓名 | 必填，必须和学生档案姓名一致 |
| 设置密码 | 必填，至少 6 位 |
| 确认密码 | 前端本地校验，两次密码一致后再提交 |

按钮：

```text
注册
```

请求接口：

```http
POST /api/auth/register
```

请求体：

```json
{
  "studentNo": "20230004",
  "name": "测试学生004",
  "password": "Student123456"
}
```

注册成功后建议提示：

```text
注册成功，请使用学号和刚设置的密码登录。
```

然后自动切换回登录 Tab，并把学号填入登录账号输入框。

## 异常提示建议

| 后端状态 | 场景 | 前端提示 |
| --- | --- | --- |
| 400 | 学号和姓名不匹配 | 学号或姓名不正确，请确认后重试 |
| 400 | 学生不是在校状态 | 当前学生状态不允许注册 |
| 409 | 已经注册过 | 该学生已注册，请直接登录 |
| 401 | 登录账号或密码错误 | 账号或密码错误 |

## 给前端同学的一段说明

登录注册页请做成同一个认证页面，顶部用 Tab 在“登录”和“学生注册”之间切换。登录时账号字段统一叫 `username`，学生注册后使用学号作为登录账号；注册时不要让用户选择角色，只展示“学号、姓名、设置密码、确认密码”四个字段。注册接口只提交 `studentNo`、`name`、`password`，后端会自动判断这个学号和姓名是否存在于学生档案，并且只会创建 `student` 角色账号。注册成功后不要直接进入系统，提示“注册成功，请使用学号和刚设置的密码登录”，然后切换回登录页。登录成功后保存后端返回的 `data.token` 和 `data.user`，后续请求在 Header 中携带 `Authorization: Bearer <token>`。
