# 课程详情接口

## 查询课程详情及安排

`GET /api/base-data/courses/:id`

可选查询参数：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `semesterId` | 正整数 | 仅返回该学期的考试与教材征订安排 |

详情由现有数据库关系聚合，`arrangements` 包含：

- `exams`：考试学期、班级、教室、起止时间和监考教师；
- `scheduleChanges`：调课教师、班级、课时、原因和审批状态；
- `textbookOrders`：教材、征订数量和状态；
- `teachingLogs`：授课教师、班级、日期、教学内容、考勤摘要和状态。

成功响应中的 `data` 示例：

```json
{
  "id": 1,
  "code": "CS101",
  "name": "数据库原理",
  "credits": 3.5,
  "courseType": "必修",
  "department": { "id": 2, "code": "CS", "name": "计算机系" },
  "director": {
    "id": 3,
    "employeeNo": "T003",
    "name": "张老师",
    "title": "副教授"
  },
  "arrangements": {
    "exams": [],
    "scheduleChanges": [],
    "textbookOrders": [],
    "teachingLogs": []
  }
}
```

课程不存在时返回 HTTP 404；`id` 或 `semesterId` 不是正整数时返回 HTTP 400。

说明：当前数据库没有常规课表（周次、星期、节次）实体，因此本接口只展示数据库中已经存在的课程业务安排，不虚构常规上课时间。
