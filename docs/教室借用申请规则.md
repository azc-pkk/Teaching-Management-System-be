# 教室借用申请后端规则

## 身份与权限

- `applicantId` 统一表示 `sys_user.id`，不是 `teacherId` 或 `studentId`。
- 创建申请时以后端解析 Bearer Token 得到的用户为申请人。兼容传入 `applicantId`，但它必须与登录用户一致。
- 只有 `TEACHER`、`STUDENT` 可以创建申请。
- 只有申请人本人可以修改或删除草稿、提交草稿、取消待审批申请。
- 只有 `ACADEMIC`（教务管理员）可以批准或驳回；`DEPARTMENT_ADMIN` 没有审批权限。

## 状态流转

- `DRAFT -> PENDING`：申请人提交。
- `PENDING -> APPROVED`：教务管理员批准。
- `PENDING -> REJECTED`：教务管理员驳回，`comment` 必填且不能全为空白。
- `PENDING -> CANCELLED`：申请人在开始前取消。
- 已批准申请不可取消；已经开始的申请不可取消。
- 只有 `DRAFT` 可以编辑或删除，状态不能通过普通修改接口直接变更。

## 业务校验

- `participantCount` 必须为正整数，且不能超过教室容量。
- 只有 `AVAILABLE` 教室允许申请；`OCCUPIED`、`MAINTENANCE` 直接排除。
- 开始时间必须早于结束时间，且开始时间必须在未来。
- 教师和学生最多提前 7 天申请。
- `DRAFT`、`PENDING`、`APPROVED` 申请参与时间冲突判断。
- 同一教室的重叠时段不可重复申请，并与考试占用时间一起校验。
- 同一申请人在同一时段最多有一条有效申请。

## 审批记录与响应

提交、批准、驳回和取消都会写入 `approval_record`。申请列表和详情返回：

- `latestApproval`：最新操作、操作人、时间和意见。
- `approvalHistory`：按时间升序排列的完整审批时间线。

## 业务错误码

| 错误码                       | 含义                           |
| ---------------------------- | ------------------------------ |
| `APPLICANT_MISMATCH`         | 请求中的申请人与当前用户不一致 |
| `APPLICANT_ROLE_FORBIDDEN`   | 当前角色不能发起申请           |
| `APPROVAL_FORBIDDEN`         | 当前角色无审批权限             |
| `REQUEST_OWNER_REQUIRED`     | 当前用户不是申请人             |
| `REJECTION_COMMENT_REQUIRED` | 驳回意见缺失                   |
| `INVALID_STATUS_TRANSITION`  | 状态流转不合法                 |
| `REQUEST_NOT_EDITABLE`       | 当前状态不可编辑               |
| `REQUEST_NOT_DELETABLE`      | 当前状态不可删除               |
| `REQUEST_ALREADY_STARTED`    | 申请已开始，不能取消           |
| `INVALID_TIME_RANGE`         | 开始、结束时间顺序错误         |
| `PAST_START_TIME`            | 申请时间已过去                 |
| `ADVANCE_LIMIT_EXCEEDED`     | 超出普通申请提前 7 天限制      |
| `CAPACITY_EXCEEDED`          | 使用人数超过教室容量           |
| `CLASSROOM_NOT_AVAILABLE`    | 教室状态不可申请               |
| `CLASSROOM_TIME_CONFLICT`    | 教室时间冲突                   |
| `APPLICANT_TIME_CONFLICT`    | 申请人已有重叠申请             |
