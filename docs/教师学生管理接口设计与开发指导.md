# 教师管理与学生管理接口设计与开发指导

本文档用于指导后端开发员实现“教师管理”和“学生管理”两个基础数据模块，并作为前端同学对接、项目经理验收、Apifox 调试的统一依据。

对应当前数据库模型：

- 教师表：`teacher`
- 学生表：`student`
- 院系表：`department`
- 班级表：`class_group`

## 1. 功能范围

### 1.1 教师管理

用于维护教师基础信息。

核心字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | number | 教师 ID |
| `employeeNo` | string | 教师工号，唯一 |
| `name` | string | 教师姓名 |
| `departmentId` | number \| null | 所属院系 ID |
| `departmentName` | string \| null | 所属院系名称，列表展示用 |
| `teacherType` | string \| null | 教师类型，如专任教师、外聘教师 |
| `title` | string \| null | 职称 |
| `phone` | string \| null | 联系电话 |

### 1.2 学生管理

用于维护学生基础信息。

核心字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | number | 学生 ID |
| `studentNo` | string | 学号，唯一 |
| `name` | string | 学生姓名 |
| `classGroupId` | number | 班级 ID |
| `classGroupName` | string | 班级名称，列表展示用 |
| `grade` | number | 年级 |
| `status` | string | 学籍状态 |
| `phone` | string \| null | 联系电话 |

学生状态枚举：

```ts
type StudentStatus = "ENROLLED" | "SUSPENDED" | "GRADUATED" | "WITHDRAWN";
```

## 2. 权限边界

当前实习项目先按最小可落地权限设计。

| 角色 | 教师管理 | 学生管理 |
| --- | --- | --- |
| `admin` | 全部增删改查 | 全部增删改查 |
| `department_admin` | 查看和维护本院系教师 | 查看和维护本院系学生 |
| `teacher` | 只读必要信息 | 只读必要信息 |
| `student` | 不开放管理能力 | 不开放管理能力 |

后端必须校验权限，前端隐藏菜单只是辅助，不能作为安全依据。

## 3. 统一接口规范

### 3.1 Base URL

本地开发：

```text
http://localhost:3001/api
```

局域网联调时将 `localhost` 换成后端电脑 IP。

### 3.2 认证方式

登录成功后，前端拿到：

```json
{
  "success": true,
  "data": {
    "token": "...",
    "user": {}
  }
}
```

后续管理接口请求头统一携带：

```http
Authorization: Bearer <token>
```

### 3.3 成功响应

```ts
type ApiResponse<T> = {
  success: true;
  data: T;
};
```

示例：

```json
{
  "success": true,
  "data": {
    "id": 1,
    "name": "张伟"
  }
}
```

### 3.4 失败响应

```ts
type ApiErrorResponse = {
  success: false;
  error: {
    code: string;
    message: string;
  };
};
```

示例：

```json
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "教师工号不能为空"
  }
}
```

### 3.5 分页响应

```ts
type PageResult<T> = {
  list: T[];
  page: number;
  pageSize: number;
  total: number;
};
```

## 4. 教师管理 API

### 4.1 查询教师列表

```http
GET /api/teachers
```

Query 参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `page` | number | 否 | 页码，默认 1 |
| `pageSize` | number | 否 | 每页数量，默认 10 |
| `keyword` | string | 否 | 搜索工号、姓名 |
| `departmentId` | number | 否 | 院系 ID |
| `teacherType` | string | 否 | 教师类型 |
| `title` | string | 否 | 职称 |

响应：

```json
{
  "success": true,
  "data": {
    "list": [
      {
        "id": 1,
        "employeeNo": "T0001",
        "name": "张伟",
        "departmentId": 2,
        "departmentName": "计算机系",
        "teacherType": "专任教师",
        "title": "副教授",
        "phone": "13800000001"
      }
    ],
    "page": 1,
    "pageSize": 10,
    "total": 1
  }
}
```

### 4.2 查询教师详情

```http
GET /api/teachers/:id
```

### 4.3 新增教师

```http
POST /api/teachers
```

Body：

```json
{
  "employeeNo": "T0100",
  "name": "李老师",
  "departmentId": 2,
  "teacherType": "专任教师",
  "title": "讲师",
  "phone": "13800001000"
}
```

校验要求：

- `employeeNo` 必填且唯一。
- `name` 必填。
- `departmentId` 如填写，必须存在于 `department` 表。
- `phone` 如填写，建议校验手机号格式。

### 4.4 修改教师

```http
PATCH /api/teachers/:id
```

Body 可传部分字段：

```json
{
  "name": "李老师",
  "departmentId": 3,
  "teacherType": "外聘教师",
  "title": "教授",
  "phone": "13800001001"
}
```

### 4.5 删除教师

```http
DELETE /api/teachers/:id
```

建议第一阶段使用物理删除；如果教师已被课程、考试、调课、日志等业务引用，应返回业务错误，不允许删除。

错误示例：

```json
{
  "success": false,
  "error": {
    "code": "TEACHER_IN_USE",
    "message": "该教师已被业务数据引用，不能删除"
  }
}
```

## 5. 学生管理 API

### 5.1 查询学生列表

```http
GET /api/students
```

Query 参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `page` | number | 否 | 页码，默认 1 |
| `pageSize` | number | 否 | 每页数量，默认 10 |
| `keyword` | string | 否 | 搜索学号、姓名 |
| `classGroupId` | number | 否 | 班级 ID |
| `grade` | number | 否 | 年级 |
| `status` | string | 否 | 学籍状态 |

响应：

```json
{
  "success": true,
  "data": {
    "list": [
      {
        "id": 1,
        "studentNo": "S20260001",
        "name": "王同学",
        "classGroupId": 1,
        "classGroupName": "软件工程 2601 班",
        "grade": 2026,
        "status": "ENROLLED",
        "phone": "13900000001"
      }
    ],
    "page": 1,
    "pageSize": 10,
    "total": 1
  }
}
```

### 5.2 查询学生详情

```http
GET /api/students/:id
```

### 5.3 新增学生

```http
POST /api/students
```

Body：

```json
{
  "studentNo": "S20260100",
  "name": "赵同学",
  "classGroupId": 1,
  "grade": 2026,
  "status": "ENROLLED",
  "phone": "13900000100"
}
```

校验要求：

- `studentNo` 必填且唯一。
- `name` 必填。
- `classGroupId` 必填且必须存在于 `class_group` 表。
- `grade` 必填，建议为四位年份。
- `status` 必须是 `ENROLLED`、`SUSPENDED`、`GRADUATED`、`WITHDRAWN` 之一。

### 5.4 修改学生

```http
PATCH /api/students/:id
```

Body 可传部分字段：

```json
{
  "name": "赵同学",
  "classGroupId": 2,
  "grade": 2026,
  "status": "SUSPENDED",
  "phone": "13900000101"
}
```

### 5.5 删除学生

```http
DELETE /api/students/:id
```

如果学生已有毕业审核、考试记录等业务数据，应返回业务错误，不允许删除。

错误示例：

```json
{
  "success": false,
  "error": {
    "code": "STUDENT_IN_USE",
    "message": "该学生已被业务数据引用，不能删除"
  }
}
```

## 6. 下拉选项接口

为了方便前端表单选择院系和班级，建议补充两个只读接口。

### 6.1 院系列表

```http
GET /api/base-data/departments
```

响应：

```json
{
  "success": true,
  "data": [
    {
      "id": 1,
      "name": "信息工程学院",
      "code": "D001",
      "type": "COLLEGE",
      "parentId": null
    }
  ]
}
```

### 6.2 班级列表

```http
GET /api/base-data/class-groups
```

Query 参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `departmentId` | number | 否 | 按院系过滤 |
| `majorId` | number | 否 | 按专业过滤 |
| `grade` | number | 否 | 按年级过滤 |

响应：

```json
{
  "success": true,
  "data": [
    {
      "id": 1,
      "name": "软件工程 2601 班",
      "grade": 2026,
      "majorId": 1,
      "majorName": "软件工程",
      "departmentId": 2,
      "departmentName": "软件工程系"
    }
  ]
}
```

## 7. 后端开发建议

建议按以下文件结构实现：

```text
src/modules/teachers/
  teachers.module.ts
  teachers.controller.ts
  teachers.service.ts
  dto/
    create-teacher.dto.ts
    update-teacher.dto.ts
    query-teacher.dto.ts

src/modules/students/
  students.module.ts
  students.controller.ts
  students.service.ts
  dto/
    create-student.dto.ts
    update-student.dto.ts
    query-student.dto.ts
```

开发顺序：

1. 先写 DTO 和参数校验。
2. 再写列表查询，支持分页和筛选。
3. 再写新增、详情、修改、删除。
4. 最后补权限校验和业务引用保护。

## 8. Apifox 使用说明

已经提供 OpenAPI 文件：

```text
docs/apifox-teacher-student.openapi.json
```

导入步骤：

1. 打开 Apifox。
2. 进入项目。
3. 点击“导入”。
4. 选择“OpenAPI/Swagger”。
5. 选择 `docs/apifox-teacher-student.openapi.json`。
6. 导入后配置环境变量：

```text
baseUrl = http://localhost:3001/api
token = 登录接口返回的 token
```

7. 在请求头中统一使用：

```http
Authorization: Bearer {{token}}
```

## 9. 验收清单

- 教师列表支持分页、关键字、院系筛选。
- 教师新增时工号唯一校验生效。
- 教师修改后列表和详情能看到最新数据。
- 教师被业务引用时不能直接删除。
- 学生列表支持分页、关键字、班级、年级、状态筛选。
- 学生新增时学号唯一校验生效。
- 学生状态只能使用枚举值。
- 学生被业务引用时不能直接删除。
- 未登录访问管理接口返回 401。
- 非授权角色访问管理接口返回 403。
