# 教室管理前端开发指导

本文档面向前端同学，用于实现“教室管理”页面。后端已提供教室 CRUD、筛选、下拉选项接口。

## 1. 页面目标

教室管理用于维护学校教室基础信息，是后续“教室申请”“考务安排”等模块的基础数据。

页面需要支持：

- 查看教室列表
- 按关键字、校区、楼栋、类型、状态筛选
- 新增教室
- 编辑教室
- 删除教室
- 查看教室详情

## 2. 页面路由建议

```text
/classrooms
```

或者放在基础数据菜单下：

```text
/base-data/classrooms
```

菜单名称：

```text
教室管理
```

## 3. 后端接口

Base URL：

```text
http://localhost:3001/api
```

### 3.1 查询教室列表

```http
GET /api/classrooms
```

Query 参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `page` | number | 否 | 页码，默认 1 |
| `pageSize` | number | 否 | 每页数量，默认 10 |
| `keyword` | string | 否 | 搜索校区、楼栋、教室号、类型 |
| `campus` | string | 否 | 校区 |
| `building` | string | 否 | 楼栋 |
| `type` | string | 否 | 教室类型 |
| `status` | string | 否 | 教室状态 |

请求示例：

```http
GET /api/classrooms?page=1&pageSize=10&keyword=A101&status=AVAILABLE
```

响应示例：

```json
{
  "success": true,
  "data": {
    "list": [
      {
        "id": 1,
        "campus": "主校区",
        "building": "A栋",
        "roomNo": "A101",
        "type": "多媒体教室",
        "capacity": 80,
        "area": 120,
        "status": "AVAILABLE"
      }
    ],
    "page": 1,
    "pageSize": 10,
    "total": 1
  }
}
```

### 3.2 查询教室下拉选项

```http
GET /api/classrooms/options
```

响应示例：

```json
{
  "success": true,
  "data": {
    "campuses": ["主校区", "东校区"],
    "buildings": ["A栋", "B栋"],
    "types": ["多媒体教室", "实验室", "阶梯教室"],
    "statuses": ["AVAILABLE", "OCCUPIED", "MAINTENANCE"]
  }
}
```

### 3.3 查询教室详情

```http
GET /api/classrooms/:id
```

### 3.4 新增教室

```http
POST /api/classrooms
```

Body：

```json
{
  "campus": "主校区",
  "building": "A栋",
  "roomNo": "A101",
  "type": "多媒体教室",
  "capacity": 80,
  "area": 120,
  "status": "AVAILABLE"
}
```

### 3.5 修改教室

```http
PATCH /api/classrooms/:id
```

Body 可传部分字段：

```json
{
  "capacity": 100,
  "status": "MAINTENANCE"
}
```

### 3.6 删除教室

```http
DELETE /api/classrooms/:id
```

如果教室已经被考试或教室申请引用，后端会返回业务冲突，前端需要提示：

```text
该教室已被业务数据引用，不能删除
```

## 4. 字段说明

| 字段 | 类型 | 页面展示 | 表单输入 |
| --- | --- | --- | --- |
| `id` | number | 不展示或作为详情字段 | 不输入 |
| `campus` | string | 校区 | 输入框/下拉 |
| `building` | string | 楼栋 | 输入框/下拉 |
| `roomNo` | string | 教室号 | 必填输入框 |
| `type` | string | 教室类型 | 输入框/下拉 |
| `capacity` | number | 容量 | 必填数字输入 |
| `area` | number \| null | 面积 | 数字输入，可空 |
| `status` | string | 状态标签 | 必填下拉 |

## 5. 状态映射

| 后端值 | 中文显示 | 建议颜色 |
| --- | --- | --- |
| `AVAILABLE` | 可用 | 绿色 |
| `OCCUPIED` | 占用 | 蓝色 |
| `MAINTENANCE` | 维护中 | 橙色 |

前端建议统一封装：

```ts
const classroomStatusMap = {
  AVAILABLE: { label: "可用", color: "green" },
  OCCUPIED: { label: "占用", color: "blue" },
  MAINTENANCE: { label: "维护中", color: "orange" },
};
```

## 7. 前端交互要求

### 7.1 页面初始化

页面加载时同时请求：

```http
GET /api/classrooms/options
GET /api/classrooms?page=1&pageSize=10
```

第一个接口用于筛选项和表单下拉，第二个接口用于列表数据。

### 7.2 查询

点击“查询”时，把当前筛选条件传给列表接口。

例如：

```ts
GET /api/classrooms?page=1&pageSize=10&campus=主校区&status=AVAILABLE
```

### 7.3 重置

点击“重置”时：

- 清空 `keyword`
- 清空 `campus`
- 清空 `building`
- 清空 `type`
- 清空 `status`
- 页码回到 1
- 重新请求列表

### 7.4 新增

点击“新增教室”打开弹窗。

前端校验：

- `roomNo` 必填
- `capacity` 必填且大于 0
- `area` 如果填写，必须大于等于 0
- `status` 默认 `AVAILABLE`

保存成功后：

- 关闭弹窗
- 提示“新增成功”
- 重新请求当前列表

### 7.5 编辑

点击“编辑”打开弹窗，回填当前行数据。

保存成功后：

- 关闭弹窗
- 提示“修改成功”
- 重新请求当前列表

### 7.6 删除

点击“删除”时需要二次确认：

```text
确定删除教室 A101 吗？删除后不可恢复。
```

删除成功后：

- 提示“删除成功”
- 重新请求当前列表

如果后端返回 `409`：

```text
该教室已被业务数据引用，不能删除
```

## 8. TypeScript 类型建议

```ts
type ClassroomStatus = "AVAILABLE" | "OCCUPIED" | "MAINTENANCE";

type Classroom = {
  id: number;
  campus: string | null;
  building: string | null;
  roomNo: string;
  type: string | null;
  capacity: number;
  area: number | null;
  status: ClassroomStatus;
};

type ClassroomOptions = {
  campuses: string[];
  buildings: string[];
  types: string[];
  statuses: ClassroomStatus[];
};

type ClassroomQuery = {
  page: number;
  pageSize: number;
  keyword?: string;
  campus?: string;
  building?: string;
  type?: string;
  status?: ClassroomStatus;
};
```

## 9. 验收清单

- 进入页面能看到教室列表。
- 筛选项能正常加载校区、楼栋、类型、状态。
- 关键字能搜索教室号、楼栋、校区、类型。
- 新增教室后列表能看到新数据。
- 重复教室号新增时能提示失败。
- 修改容量、面积、状态后能立即刷新列表。
- 删除未被引用的教室成功。
- 删除已被业务引用的教室时有明确错误提示。
- 分页能正常切换。
